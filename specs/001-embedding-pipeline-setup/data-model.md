# Data Model: Embedding Pipeline

## Document

Represents a unit of content extracted from the website (e.g., a page, section, or chapter) with its text content, metadata, and URL

**Fields**:
- `id`: Unique identifier for the document
- `url`: The source URL of the document
- `title`: Title of the document
- `content`: The extracted text content
- `section`: The section or chapter the document belongs to
- `created_date`: Timestamp when the document was extracted
- `updated_date`: Timestamp when the document was last updated
- `checksum`: Content hash to detect changes

## Embedding

A vector representation of document content generated by the Cohere model, associated with its source document

**Fields**:
- `id`: Unique identifier for the embedding
- `document_id`: Reference to the source document
- `vector`: The embedding vector (1024-dimensional float array)
- `created_date`: Timestamp when the embedding was generated

## Metadata

Information about the source document including URL, title, section, creation date, and any other relevant indexing information

**Fields**:
- `document_id`: Reference to the document
- `url`: The source URL
- `title`: Title of the document
- `section`: The section or chapter
- `source_type`: Type of source (e.g., "docusaurus")
- `additional_metadata`: JSON object with any additional metadata

## Chunk

A portion of a document that fits within the token limits of the embedding model

**Fields**:
- `id`: Unique identifier for the chunk
- `document_id`: Reference to the source document
- `content`: The text content of the chunk
- `chunk_index`: Position of this chunk within the document
- `token_count`: Number of tokens in the chunk
- `checksum`: Content hash to detect changes

## Pipeline Job

A unit of work representing the processing of a website URL through the entire pipeline from ingestion to storage

**Fields**:
- `id`: Unique identifier for the job
- `url`: The source URL to process
- `status`: Current status (e.g., "pending", "in_progress", "completed", "failed")
- `created_date`: Timestamp when the job was created
- `started_date`: Timestamp when processing started
- `completed_date`: Timestamp when processing completed
- `error_message`: Any error message if the job failed
- `processed_documents_count`: Number of documents successfully processed
